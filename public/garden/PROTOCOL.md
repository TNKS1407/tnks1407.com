# 畑の計器盤 — シリアルプロトコル v1

状態: **確定**（2026-07-14）。ファームウェアはこの契約に従って実装する。
ビューア（`dashboard/index.html`）はこの仕様で既に実装済み・デモ検証済み。

## 物理層

- USB CDC（Pico の USB シリアル）。115200 bps 8N1（CDC なのでボーレートは形式的だが、ビューアは 115200 で open する）
- テキスト行プロトコル。**1行 = 1メッセージ、改行 LF（`\n`）、UTF-8**
- `#` で始まる行はデバッグ/ログ用。ビューアは無視する（FW は自由に使ってよい）

## テレメトリ（Pico → ブラウザ）

1行の JSON。例:

```
{"t":123456,"soil":[41520,38900,null],"temp":24.6,"pump":0}
```

| フィールド | 型 | 意味 |
|---|---|---|
| `t` | number | 起動からの経過 ms（`millis()`）。uint32 なので約49.7日でラップする。**絶対時刻ではない**。履歴の時刻はブラウザが受信時の時計で付与する（Pico に RTC は無い前提） |
| `soil` | array | **長さ3固定**。ch0=GP26/A0, ch1=GP27/A1, ch2=GP28/A2。値は 0–65535 の生ADC。RP2040 の ADC は12bit なので FW は `読値 << 4` で正規化して送る（実質最大 65520）。**未接続チャネルは `null`** |
| `temp` | number \| null | 気温 ℃（float）。センサ未搭載・読み取り失敗は `null`。センサ種別は未定（DS18B20 等、FW 側の自由） |
| `pump` | 0 \| 1 | 給水ポンプの**実際の駆動出力**の状態（コマンドのエコーではない） |

- 送信周期: **1 Hz 推奨**（許容 0.5〜2 s）。ポンプ状態が変化した瞬間は周期を待たず1行送ってよい
- 未接続の判定方法は FW の自由（推奨: 実装が簡単な「FW 設定でチャネル有効/無効を固定」）。契約は「未接続なら `null`」のみ
- 受信側（ビューア）の規則: **未知のフィールドは無視**（前方互換のため FW は将来フィールドを足してよい）。パースできない行は捨ててエラー計上のみ

## コマンド（ブラウザ → Pico）

1行の JSON + LF。

給水 ON:

```
{"cmd":"pump","on":true,"sec":5}
```

- `sec`: 給水秒数。**1〜10**。`on:true` のとき必須。範囲外は FW がクランプ

即時停止:

```
{"cmd":"pump","on":false}
```

- 応答: **専用の ACK は無い**。次のテレメトリの `pump` フィールドが真実（ビューアはそれで確認する）
- 未知の `cmd`・壊れた JSON: 無視する（`#` 行でログしてよい）

## 安全規約（FW は必ず守る）

1. **ポンプ連続 ON は最大 10 秒**。FW 内タイマーで強制 OFF する。コマンドが何を要求しても超えない。**ON 中の再コマンドで延長しない**（いったん OFF を経由する）
2. OFF 後 **5 秒間はクールダウン**とし、次の ON コマンドを受け付けない（拒否したことを `#` 行で通知してよい）
3. 起動直後・リセット直後はポンプ OFF
4. パース不能な行で状態を変えない（暴走しない）
5. 自動給水ロジックは v1 では FW に置かない。FW は手動コマンドの実行のみ。しきい値判定（「乾き」線）はブラウザ側の表示・将来の自動化判断に使う（どちら側に置くかは v2 で決める）

## セッション例

```
（Pico 起動）
# garden-fw v1 boot
{"t":1024,"soil":[52310,50790,null],"temp":24.1,"pump":0}
{"t":2025,"soil":[52288,50801,null],"temp":24.1,"pump":0}
（← ブラウザが送信: {"cmd":"pump","on":true,"sec":5}）
{"t":2310,"soil":[52290,50777,null],"temp":24.1,"pump":1}   ← 変化時は即時1行
{"t":3025,"soil":[52101,50530,null],"temp":24.2,"pump":1}
...
{"t":7311,"soil":[48900,46200,null],"temp":24.2,"pump":0}   ← 5秒経過で OFF
```

## 参考（契約外・ビューア側の扱い）

- **生ADCのまま送る**のが契約。%への変換はブラウザ側の2点校正（「乾いた土」「水没」の生値を取得）で行う。履歴も raw で保存するので、**校正をやり直すと過去のグラフごと引き直される**
- ビューアの localStorage: 履歴は1分間隔に間引いて7日分のリングバッファ（約400KB）。しきい値（乾き/過湿）は実機・デモ共通、校正と履歴は実機のみ永続（デモは揮発）
